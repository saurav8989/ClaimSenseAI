========================================================================
                   CLAIMSENSE AI - PROJECT FOLDER STRUCTURE
========================================================================

To keep development organized and prevent conflicts, we will use a unified
directory structure. Below is the layout for our two stack options. 

We recommend **Option A (Next.js Fullstack)** because it runs everything in 
one place and simplifies parallel development.

------------------------------------------------------------------------
OPTION A: NEXT.JS FULLSTACK (RECOMMENDED)
------------------------------------------------------------------------

All code exists in a single project tree. Developers work in designated
folders based on their roles.

ClaimSenseAI/
├── src/
│   ├── app/                      # Page views and API routes (Next.js App Router)
│   │   ├── page.jsx              # Landing / Role Selection Page
│   │   │
│   │   ├── doctor/               # [DEV 1] Doctor Note / Claim Builder Page
│   │   │   └── page.jsx
│   │   │
│   │   ├── reviewer/             # [DEV 3] Reviewer Queue & Details Page
│   │   │   └── page.jsx
│   │   │
│   │   └── api/                  # API Endpoints
│   │       ├── openimis/         # [DEV 1] Mock openIMIS API (claim storage)
│   │       │   └── route.js
│   │       ├── suggest-icd/      # [DEV 2] Autocomplete suggestions endpoint
│   │       │   └── route.js
│   │       └── analyze/          # [DEV 2] Compliance & Risk scoring endpoint
│   │           └── route.js
│   │
│   ├── components/               # Reusable Frontend Components
│   │   ├── ClaimBuilder.jsx      # [DEV 1] Form components
│   │   ├── ReviewerQueue.jsx     # [DEV 3] Main reviewer queue list
│   │   ├── DeviationTimeline.jsx # [DEV 3] Interactive STP deviation timeline
│   │   └── ui/                   # Shared UI primitives (buttons, modals)
│   │
│   ├── lib/                      # [DEV 2] Core Business Logic (No UI code)
│   │   ├── icdSuggester.js       # ICD recommendation keyword engine
│   │   ├── stpEngine.js          # Rules engine for standard treatment protocols
│   │   ├── riskScorer.js         # Score calculator and flag trigger
│   │   └── mockDb.js             # Mock SQLite or JSON claim database
│   │
│   └── styles/                   # Styling systems (Tailwind/CSS)
│
├── public/                       # Static assets (images, logos)
├── package.json
└── README.md

------------------------------------------------------------------------
MEMBER ACTION ITEM
------------------------------------------------------------------------

Once the team selects Option A or Option B:
1. **Initialize the workspace structure** (e.g. create folders `/src/app/`, `/src/components/`, and `/src/lib/` for Option A).
2. **Develop exclusively in your module directories** to prevent merge conflicts.
3. **Commit often** to the `dev` branch.
